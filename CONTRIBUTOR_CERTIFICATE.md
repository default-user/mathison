# Contributor Certificate of Origin

## Purpose

This certificate ensures that all contributors have the legal right to contribute their code to Mathison.

## Certificate Template

By submitting a contribution to this project, you certify that:

1. **Ownership:** You have the legal right to submit this contribution under the Apache License 2.0.

2. **Original Work:** The contribution is either:
   - Your original creation, OR
   - Based on open-source work with compatible licensing, with proper attribution

3. **No Proprietary Code:** You have not copied proprietary code from other projects, employers, or clients.

4. **AI-Assisted Work (if applicable):**
   - You have reviewed all AI-generated code before submission
   - You understand the contribution and can maintain it
   - You take responsibility for the correctness of AI-assisted code

5. **License Agreement:** You agree to license your contribution under Apache License 2.0.

6. **Public Record:** You understand this contribution becomes part of the public record.

## How to Attest

### Option 1: Inline Attestation (Preferred for Small Contributions)

Add to your pull request description:

```
I certify that this contribution complies with the Contributor Certificate
as specified in CONTRIBUTOR_CERTIFICATE.md.

Signed: [Your Name] <your.email@example.com>
Date: YYYY-MM-DD
```

### Option 2: Signed Certificate (For Large Contributions or Initial Setup)

Create a file named `CONTRIBUTOR_ATTESTATION_<your-github-username>.txt` in the root directory:

```
Contributor Certificate of Origin

I, [Full Legal Name], certify that my contributions to the Mathison project
comply with all terms in CONTRIBUTOR_CERTIFICATE.md.

I confirm:
- I have the legal right to contribute this code
- This code is my original work or properly attributed open-source work
- I have not copied proprietary code
- All AI-assisted contributions have been reviewed by me
- I agree to license contributions under Apache License 2.0

GitHub Username: [username]
Email: [email@example.com]
Date: [YYYY-MM-DD]

Signature: [Your Name]
```

Submit this file via pull request.

## Employer Rights Notice

If you are contributing as part of employment, ensure:

1. Your employer permits you to contribute to open-source projects
2. Your employer does not claim ownership of work done on personal time (if applicable)
3. You are not contributing company proprietary code or trade secrets

If in doubt, obtain written permission from your employer before contributing.

## What NOT to Contribute

- **Proprietary Code:** Code owned by your employer or other entities
- **Copyleft Code:** GPL, AGPL, or other copyleft-licensed code (incompatible with Apache 2.0)
- **Unverified AI Output:** AI-generated code you haven't reviewed or don't understand
- **Credentials:** API keys, passwords, private keys, or other secrets
- **Sensitive Data:** PII, confidential business information, or data you don't own

## Revocation

If you later discover your contribution violates this certificate:

1. Immediately notify the maintainers via GitHub issue (mark as SECURITY)
2. Provide details about the violation
3. The maintainers will work with you to remove or replace the code

## Legal Effect

This certificate is based on the Developer Certificate of Origin (DCO) used by the Linux kernel and other open-source projects. It provides a lightweight attestation that does not require copyright assignment.

By contributing, you agree that your contributions are irrevocably submitted under the Apache License 2.0.

## Questions

For questions about contributor attestation: open an issue in the repository or contact the maintainers.
